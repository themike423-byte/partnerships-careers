## Quick Test Accounts

For rapid testing, use these patterns:

### Pattern 1: Mail.com domains
- `test1@post.com`
- `test2@usa.com`
- `test3@email.com`
- `admin@inbox.com`
- `user@mail.com`

### Pattern 2: Use subdomains or variations
If you have access to a domain, you can use:
- `test1@yourdomain.com`
- `test2@yourdomain.com`
- `admin@yourdomain.com`

## Testing Workflow

1. Sign up at mail.com with a new domain (e.g., `post.com`)
2. Create account in your app with that email
3. First user becomes admin automatically
4. Second user from same domain joins as member (not admin)

## Note

The code blocks these personal domains:
- gmail.com, yahoo.com, hotmail.com, outlook.com, icloud.com
- mail.com, protonmail.com, aol.com, live.com, msn.com, ymail.com, zoho.com

So `@post.com`, `@usa.com`, `@email.com`, `@inbox.com` should all work!
